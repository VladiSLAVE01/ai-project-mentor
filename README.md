# 🤖 AI Project Mentor — ИИ-ассистент по управлению проектами

**AI Project Mentor** — это интеллектуальный ассистент для студентов и специалистов, который помогает разбираться в методологиях управления IT-проектами (Agile, Scrum, DevOps) и инженерных практиках. Система построена на технологии **RAG (Retrieval-Augmented Generation)**: она ищет ответы в вашей базе знаний и генерирует точные ответы с помощью LLM, строго ссылаясь на источники.

Ассистент доступен пока только в одном формате:
- **Telegram бот** — для быстрого доступа с любого устройства.

---

## ✨ Возможности

- 📚 **Ответы на основе знаний** — только по загруженным материалам (Scrum, DevOps, ТЗ и др.)
- 🔍 **Семантический поиск** — поиск релевантных фрагментов с помощью векторной базы данных (ChromaDB)
- 🧠 **Генерация ответов** — через LLM (OpenRouter / OpenAI / локально)
- 📎 **Ссылки на источники** — каждый ответ сопровождается списком документов с релевантностью
- 🎛 **Два режима работы** в Telegram:
  - *Режим базы знаний* — только по вашим материалам
  - *Общий режим* — как обычный ИИ-помощник
- 📊 **Статистика базы знаний** — количество загруженных документов

## 🚀 Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/ai-project-mentor.git
cd ai-project-mentor
```

### 2. Установите зависимости
```bash
pip install -r requirements.txt
```

### 3. Настройте переменные окружения
Создайте файл .env в корне проекта:
```bash
# Telegram
TELEGRAM_TOKEN=your_telegram_bot_token

# OpenRouter (рекомендуется для бесплатного доступа)
OPENAI_API_KEY=sk-or-v1-xxxxxxx
OPENAI_BASE_URL=https://openrouter.ai/api/v1
OPENAI_MODEL=openai/gpt-3.5-turbo

# Или для OpenAI (если доступно)
OPENAI_API_KEY=sk-xxxxxxx
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-3.5-turbo
```

### 4. Подготовьте базу знаний
Положите ваши .txt файлы в папки внутри data/raw.

### 5. Проиндексируйте документы
Запустив файл reindex.py

### 6. Запустите Telegram бота
python src/interfaces/telegram_rag_bot.py

## Как это работает
Индексация (подготовка базы знаний)
DocumentLoader загружает .txt файлы из data/raw

TextSplitter разбивает текст на чанки (chunk_size=500, overlap=50)

EmbeddingProvider превращает чанки в векторы (модель multilingual-e5-small, 384 числа)

VectorStore сохраняет векторы в ChromaDB (data/vectors)

Ответ на вопрос пользователя (RAG)
Вопрос пользователя → вектор через ту же модель эмбеддингов

Поиск похожих векторов в ChromaDB (фильтрация по релевантности ≥ 0.7)

Формирование контекста из найденных чанков

Построение промпта: system prompt + контекст + вопрос

Генерация ответа через LLM (OpenRouter)

Добавление списка источников с релевантностью

## Пример работы
```bash
📖 На основе базы знаний:

DevOps (Development + Operations) — это набор практик, объединяющих разработку и IT-операции. 
Основные принципы включают CI/CD, автоматизацию, мониторинг и IaC.

---

📚 Источники информации:
1. devopsbasics.txt (категория: devops, релевантность: 0.87)
2. agile.txt (категория: devops, релевантность: 0.82)

✅ Найдено 2 релевантных источника
```

### Параметры RAG

| Параметр | Значение |
|:---------|:---------|
| Размер чанка (`chunk_size`) | `500` символов |
| Перекрытие чанков (`chunk_overlap`) | `50` символов |
| Порог релевантности (`relevance_threshold`) | `0.7` (от 0 до 1) |
| Количество источников (`k`) | `3` |
| Модель эмбеддингов | `intfloat/multilingual-e5-small` (размерность 384) |
| Векторная БД | ChromaDB (`data/vectors`) |
| Коллекция | `project_knowledge` |
| LLM провайдер | OpenRouter / OpenAI / Ollama |
| Модель LLM | `gpt-3.5-turbo` или `google/gemini-2.0-flash-lite-preview-02-05:free` |
